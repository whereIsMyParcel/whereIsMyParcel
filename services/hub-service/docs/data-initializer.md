# Hub 초기 데이터 적재 및 마무리 (Step 10)

## 1. 구현 내용
- **HubDataInitializer**: `ApplicationRunner`를 통해 애플리케이션 기동 시 전국 17개 물류 거점 허브 데이터를 자동으로 적재합니다.
- **데이터 구성**: 서울, 부산, 대구, 인천 등 전국 주요 광역시 및 도 단위의 핵심 센터 정보(이름, 주소, 위경도)를 포함합니다.

## 2. 왜 이 방식을 선택했는가 (설계 의도)
- **운영 편의성**: MSA 환경에서 새로운 인스턴스가 뜰 때마다 수동으로 SQL을 실행할 필요 없이, 소스 코드 레벨에서 기본 인프라(허브망)가 즉시 구축되도록 했습니다.
- **다익스트라 기반 마련**: 17개 거점이 DB에 미리 존재해야만 타 서비스에서 즉시 최단 경로 탐색 기능을 테스트할 수 있으므로 필수적인 선행 단계로 정의했습니다.

## 3. 적재 데이터 상세 (17개 거점)
- 서울특별시, 부산광역시, 대구광역시, 인천광역시, 광주광역시, 대전광역시, 울산광역시, 세종특별자치시
- 경기도, 강원특별자치도, 충청북도, 충청남도, 전북특별자치도, 전라남도, 경상북도, 경상남도, 제주특별자치도

## 4. 주의사항 및 한계
- 현재 로직은 `hubRepository.count() == 0`일 때만 작동합니다. 특정 허브 정보만 업데이트하고 싶은 경우 `PATCH /api/v1/hubs/{id}` API를 사용해야 합니다.

## 5. 팀원 공유 사항
- 로컬 개발 환경 구성 후 서버를 켜기만 하면 즉시 허브를 기반으로 한 배송 테스트가 가능합니다.
- 초기 데이터가 마음에 들지 않아 초기화하고 싶다면 `p_hubs` 테이블의 데이터를 모두 지우고 다시 작성하시면 됩니다
